  • 区别

    Sys账户的权限比System多。

    OracleXE安装完后，不自带普通用户。

  8. 分组查询

    • 使用分组后，只允许查询分组字段和多行函数。

    • where后不允许出现多行函数。

    • where和having都可以使用普通字段直接进行筛选，但是where的效率高于having，而多行函数只能用使用having。

    • where执行顺序：

      from—>where—>group by —> select—>order by

    • having执行顺序

      from—>where—>group by —> select—>having—>order by

  9. Oracle内置函数

    一般使用方法：select 字段名, 函数名（字段名）,字段名 …..

    • 单行函数

      • 字符函数

        | 函数 | 功能 |
        | ———————————— | ————– |
        | INITCAP(char) | 首字母大写 |
        | LOWER(char) | 转换为小写 |
        | UPPER(char) | 转换为大写 |
        | LTRIM(char,set) | 左裁剪 |
        | RTRIM(char,set) | 右裁剪 |
        | REPLACE(char,search_str,replace_str) | 字符串替换 |
        | INSTR(char,substr[,pos]) | 查找字串的位置 |
        | SUBSTR(char,pos,len) | 获取子串 |
        | CONCAT(char1,char2) | 连接字串 |
        | LENGTH(char) | 获取字符串长度 |

      特点：

      • 不改变真实数据，只对数据做进一步修饰和处理显示。
        • 可以和字段混合使用。

    • 数值函数

  • 日期函数

    sysdate 获取系统时间

    ROUND：函数以一个星期的中间为分界点，对日期四舍五入，舍入到日期所在周的周日或者上周的周日。

  • 多行函数（即从多行记录中搜索一个结果）

    注意：多行函数不能和字段直接混用，除非分组。也不能和单行函数混用，除非分组。

函数 功能
max(字段名) 返回该字段最大值
min(字段名) 返回该字段最小值
sum(字段名) 返回该字段总和
avg(字段名) 返回该字段平均值
count(*) 查询所有记录数
count(字段名) 忽略值为null的值
count(distinct 字段名) 去重计数

  • 转换函数

    • to_number(数值型的字符串)：将字符串转换为数字。

    • to_char(数值或者日期)：将数值或者日期转换为字符。

      number—>char，指定格式：9代表占位符，例如999，999，999（占位符要长于数字位数）会按照3个一组并使用d逗号隔开。

      L代表人民币。

      • to_date(日期格式的字符)：将字符串转换为日期，指定格式’yyyy-mm-dd’。其中y是年份占位，m是月份占位，d是日占位。
      • 其他函数
        • nvl(字段名,值)：若字段名是null，返回对应的值。
        • nvl2(字段名,处理1,处理2)：若字段不为null执行处理1，否则执行处理2。
        • decode(字段名，值1，处理1，值2，处理2，值3，处理3….公共处理)：

七、子查询

where子句中可以出现查询语句，该查询语句称为子查询。

    筛选条件不明确，需要进行一次查询，并且查询 只有一个数据。

  2. 多行子查询

    子查询的结果只有一个但字段右多个，考虑使用多行子查询，即使用关键字。
